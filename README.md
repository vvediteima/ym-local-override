# YM Local Override

Кастомный Windows-клиент для Яндекс Музыки с локальной подменой аудио по `track ID`.

Идея простая:
- интерфейс, плейлисты, лайки, история, поиск и рекомендации остаются у обычной Яндекс Музыки;
- для выбранных треков клиент подменяет источник аудио на локальный файл;
- подмена хранится в локальной базе replacements.

## Для пользователя

Если вы просто хотите скачать и запустить готовую версию, Python не нужен.

1. Скачайте репозиторий целиком как ZIP.
2. Откройте папку `portable`.
3. Запустите `portable\Launch YM Local Override.cmd`.
4. Если это первый запуск, войдите в Яндекс Музыку в открывшемся окне.

В `portable` уже лежат:
- готовый launcher;
- готовый helper;
- текущая база замен;
- локальные аудиофайлы замен.

## Для разработки

Если вы хотите править проект, пересобирать `.exe` или тестировать исходники, тогда нужна локальная среда разработки.

### Подготовка

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
```

### Запуск из исходников

```powershell
.\Launch YM Local Override.cmd
```

Сейчас launcher в первую очередь использует локальный `.venv`, чтобы вы всегда запускали свежие исходники, а не старый собранный билд.

### Сборка helper

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_helper.ps1
```

### Сборка launcher

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_launcher.ps1
```

## Как добавить замену

1. Откройте нужный трек в клиенте.
2. Нажмите кнопку замены.
3. В uploader выберите один из двух вариантов:
   - `Файл с компьютера`
   - `Прямая ссылка`
4. Сохраните замену.

После этого трек будет оставаться тем же самым для Яндекс Музыки по метаданным и логике сервиса, но аудио будет играть из вашей локальной подмены.

## Структура проекта

- `app/` — helper, launcher и backend-логика.
- `ym-extension/` — расширение, которое интегрируется в web player Яндекс Музыки.
- `static/` — UI менеджера замен.
- `portable/` — готовая one-click сборка без необходимости ставить Python.
- `data/` — локальная база замен и аудиофайлы.

## Важно

- Корневые `data/uploads` и `data/replacements.json` считаются локальными пользовательскими данными.
- В репозиторий они не коммитятся.
- Для раздачи готового варианта используется папка `portable/`, в которую уже включена отдельная копия нужных данных.

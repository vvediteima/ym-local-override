# YM Local Override

Локальный helper и launcher для Windows, которые поднимают отдельный product-profile в любом поддерживаемом Chromium-браузере. Официальный web player Яндекс Музыки остаётся главным, а для выбранных `track ID` поверх него проигрывается ваш локальный аудиофайл.

## Что уже реализовано

- Локальный helper на `FastAPI` с web-интерфейсом управления заменами.
- Загрузка локального аудиофайла для конкретного `track ID`.
- Раздача сохранённых замен по локальному URL.
- Автоматический launcher продукта, который:
  - сам стартует helper;
  - сам открывает отдельный профиль браузера в app-режиме;
  - сам подгружает встроенное `ym-extension` без ручной установки.
- Расширение для встроенного product-profile, которое:
  - пытается определить текущий трек на `music.yandex.ru`;
  - спрашивает у helper, есть ли локальная замена;
  - при наличии замены глушит официальный элемент и синхронно запускает локальный файл.

## Запуск

1. Создайте виртуальное окружение и установите зависимости:

```powershell
python -m venv .venv
.\.venv\Scripts\python -m pip install -r requirements.txt
```

2. Самый простой запуск:

```powershell
.\Launch YM Local Override.cmd
```

Это запускает один отдельный продуктовый window c Яндекс Музыкой и встроенным кастомным клиентом.

Или напрямую:

```powershell
.\dist\YMLocalOverride\YMLocalOverride.exe
```

3. При первом запуске в открывшемся окне Яндекс Музыки войдите в аккаунт. Этот логин сохраняется в стабильном profile вне `dist`, поэтому не должен сбрасываться после пересборки launcher.

4. Внутри страницы появится компактная плашка `YM Custom Client`, небольшие кнопки-иконки замены рядом с треками и тихая метка замены у уже переопределённых треков.

5. Если нужно открыть helper вручную для отладки, он доступен по адресу:

```text
http://127.0.0.1:9876/
```

## Сборка helper в exe

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_helper.ps1
```

## Сборка launcher в exe

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build_launcher.ps1
```

## Ограничения текущего MVP

- Расширение использует эвристики для определения текущего трека и элемента проигрывания. Это уже рабочая база, но на отдельных экранах Яндекс Музыки селекторы могут потребовать дотюнинга.
- Официальный плеер не ломается и остаётся источником истории/контекста, но если локальная версия сильно отличается по длине от оригинала, синхронизация будет неидеальной.
- Сейчас продукт использует отдельный профиль в `.runtime\product-profile`, чтобы не конфликтовать с вашим основным браузером и не требовать ручной установки расширения.
